# Security Policy

## Supported Versions

Only the current release track receives security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0.0 | :x:                |

## Reporting a Vulnerability

We take the security of AssetFlow seriously. If you find a security issue, please do not open a public issue. Instead, report it responsibly.

### How to Report

Please email security reports to our development team or contact team members directly through Team Esc(Reality); profiles on GitHub.

Your email should include:
- A description of the vulnerability and its potential impact.
- Step-by-step instructions (or a proof-of-concept script/exploit) to reproduce the issue.
- Details of the environment used in testing (e.g., OS version, browser version, client-server configuration).

### Scope and Resolution

Once received, the team will:
1. Acknowledge receipt of the report within 48 hours.
2. Work on a fix as quickly as possible.
3. Keep the reporter informed of progress.
4. Publish a security release or patch once resolved, crediting the reporter if desired.
